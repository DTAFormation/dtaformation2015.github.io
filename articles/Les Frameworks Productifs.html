LES FRAMEWORKS PRODUCTIFS



Introduction :

Le principe des Frameworks productifs a vu le jour en 2004 avec "Ruby On Rails". L'idée maitresse de ces frameworks est de proposer un environnement cadré pour accélérer le développement des applications.

Les avantages et inconvénients :

Les frameworks productifs permettent de gagner du temps lors de la réalisation d'application à courtes durées de vie. Toutefois, il est plus difficile de configurer en détail des projets importants.

Les différents acteurs du marché:


Pour l'écosystème JAVA

Play!
Grails
SpringBoot


Pour l'écosystème Php :

Symphony
Cake Php


Pour Python :

Django


Pour l'écosytème Javascript :

Sails
Meteor
Mean.js

Conclusion :

En bref, les frameworks productifs sont particulièrements efficaces pour des projets à forts contraintes de temps ou pour développeurs pressés.
