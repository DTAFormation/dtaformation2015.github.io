L'essentiel du web

Introduction

Lors de ce chapitre nous allons aborder les principales caractéristiques du web. Nous allons nous interresser dans un premier temps nous allons nous pencher sur les caractéristiques du protocole HTTP. Ensuite nous allons aborder la communication HTTP, de la requete HTTP. Pour finir nous allons parler de la reponse HTTP

HTTP?

HTTP signifie "HyperText Transfert Protocol". C'est un protocole de communication basé sur TCP/IP et utilisé pour le Web. Le HTTP ne maintient pas de connexion entre un client et un serveur. Ce protocole est sans état (Stateless). Le HTTP est indépendant du type de données transportées et est synchrone (une requête = une réponse)

Communication HTTP

La machine dite "cliente" va envoyer une requête HTTP à la machine dite "serveur". Le serveur va traiter la demande et renvoyer la réponse HTTP adéquate au client

Requête HTTP

Le format de la requête HTTP est le suivant :
[METHODE][URI]HTTP/[VERSION]
[CHAMPS ENTETE au format "CLE:VALEUR"]
Ligne Blanche
[CORPS DE LA REQUETE(optionnel)]
Les méthodes possibles sont GET, POST, HEAD, PUT, DELETE, OPTIONS, PATCH, TRACE ...

Réponse HTTP

Il y a plusieurs statut de réponse
- 100-199 : Information
- 200-299 : Succès
- 300-399 : Redirection
- 400-499 : Erreur dans la requête
- 500-599 : Erreur côté serveur

Conclusion

En résumé le web est basé sur un protocole HTTP, protocole sans état, synchrone basé sur TCP/IP. La requête HTTP a un format spécial et il y a également un code pour la réponse HTTP. Bref le protocole HTTP c'est de la balle
